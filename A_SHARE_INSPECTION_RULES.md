## 一、A股市场时间常识（必须掌握）

### 1.1 交易时间
- **工作日**: 周一至周五（节假日除外）
- **交易时段**: 09:30-11:30, 13:00-15:00
- **数据更新**: 收盘后 15:00-17:00 逐步更新，非实时
- **周末/节假日**: 无新数据，最新数据停留在最后一个交易日

### 1.2 数据时效性判断标准
- 今天是 **交易日** → 期望数据到 T-0（当天收盘后）
- 今天是 **周末** → 最新数据到周五收盘即可，不算延迟
- 今天是 **节假日** → 最新数据到上一个交易日收盘即可，不算延迟
- 信号扫描 → 收盘后 1-2 小时运行；若周末/节假日/停跑，延迟 1-2 天正常；交易日仍应按最近交易日评估
- 定时任务 → 如果 Cron 是每日/每周，按配置判断

### 1.3 常见误区
- ❌ 错误: "今天是周六，数据只到周五，说明延迟1天"
- ✅ 正确: 周末不开市，周五数据就是最新，不算延迟
- ❌ 错误: "信号最新日期是3天前，严重滞后"
- ✅ 正确: 如果是周末/停跑，1-2天正常；第3天应结合最近交易日和任务状态判断
- ❌ 错误: "数据日期不是今天，说明系统有问题"
- ✅ 正确: 先看今天是否是交易日

## 三、A股数据特征与判断标准

### 3.1 股票代码特征
- **000XXX**: 深圳主板（如000001平安银行）
- **002XXX**: 深圳中小板（已合并到主板）
- **300XXX**: 深圳创业板
- **600XXX**: 上海主板
- **601XXX**: 上海主板（大蓝筹）
- **603XXX**: 上海主板（中小盘）
- **688XXX**: 上海科创板
- **8XX / 4XX**: 北京新三板/北交所
- **退市股**: 名称可能为空或显示代码，如000003（已退市）
- **ST股**: 名称前缀为*ST或ST，如*ST美丽

### 3.2 名称显示为代码 ≠ 一定是Bug
| 情况 | 说明 | 是否Bug |
|------|------|---------|
| 退市股名称=代码 | 该股票已退市，无标准名称 | ❌ 正常 |
| 新股名称=代码 | 上市首日可能暂无名称 | ⚠️ 可能正常 |
| 普通股名称=代码 | 如000001显示"000001" | ✅ 是Bug |
| 实时行情name=null | 接口不返回名称，需后端补全 | ⚠️ 需优化 |

### 3.3 价格数据的合理性判断
| 场景 | 合理范围 | 异常判断 |
|------|----------|----------|
| 普通A股价格 | 1-1000元 | 低于1分或超过1000需检查 |
| 高价股（茅台） | 500-2000元 | 正常 |
| 低价股 | 1-5元 | 正常，尤其是ST股 |
| ST股 | 通常低于5元 | 涨跌停按 ±5% 计算 |
| 涨停价格 | 前收盘价 × 1.10（±0.01） | 超出范围需检查 |
| 跌停价格 | 前收盘价 × 0.90（±0.01） | 超出范围需检查 |
| ST涨停价格 | 前收盘价 × 1.05（±0.01） | 超出范围需检查 |
| ST跌停价格 | 前收盘价 × 0.95（±0.01） | 超出范围需检查 |
| 跌停价格 | 前收盘价 × 0.90（±0.01） | 超出范围需检查 |
| 北交所价格 | 涨跌幅30% | 主板不同规则 |
| 科创板/创业板 | 涨跌幅20% | 主板不同规则 |

### 3.4 成交量数据的合理性判断
| 场景 | 合理范围 | 判断标准 |
|------|----------|----------|
| 大盘股日成交量 | 10亿-100亿 | 正常 |
| 小盘股日成交量 | 1000万-1亿 | 正常 |
| ST股成交量 | 可能极低 | 正常 |
| 涨停股成交量 | 可能极低（一字板） | 正常 |
| 地量 | 低于60日均量50% | 正常（缩量） |
| 天量 | 高于60日均量2-3倍 | 正常（放量） |

### 3.5 涨跌幅颜色（A股习惯）
- **红色 = 上涨**（+）
- **绿色 = 下跌**（-）
- **注意**: 与国际惯例（红跌绿涨）相反，这是A股标准
- **涨停**: 红色，+10%（或+20%科创板）
- **跌停**: 绿色，-10%（或-20%科创板）

## 四、A股市场特殊场景

### 4.1 全市场下跌时
- 当大盘指数跌幅 > 2%（如-2.26%），几乎所有板块都会下跌
- 板块涨跌幅集中在-0.01%~-5%之间是正常的
- **不能**仅凭"所有板块都跌"就判断为硬编码
- **判断方法**: 对比大盘指数涨跌幅与板块涨跌幅是否合理

### 4.2 板块数据特征
- 板块成分股列表为空 ≠ 一定是Bug
- 可能原因:
  1. 板块数据未加载（TDX板块数据缺失）
  2. 板块分类文件未更新
  3. 数据源本身不提供成分股列表
- **判断方法**: 检查 TDX 目录下是否有板块数据文件

### 4.3 信号数据延迟
- 信号通常是**收盘后扫描**生成的
- 扫描频率: 每日一次（收盘后）、每周一次（周末）
- 延迟 1-2 天是正常的，不是严重问题
- **判断方法**: 检查最近交易日日期 vs 信号日期

### 4.4 数据库为空的情况
- SQLite 数据库为空 ≠ 系统不可用
- 可能原因:
  1. 系统采用"无状态设计"，数据从 TDX 实时读取
  2. 数据库初始化脚本未运行（但首次运行可能不需要）
  3. 回测结果/信号历史等需要持久化的数据确实未保存
- **判断方法**: 检查是否影响回测结果保存、信号历史记录等核心功能

## 五、排查优先级（A股定制版）

### 5.1 🔴 Critical（真正严重）
1. 页面白屏/崩溃 → 完全无法使用
2. 核心数据乱码（股票名称、价格乱码）→ 无法识别
3. 价格数据异常（0元、负数、极端值）→ 可能误导交易决策
4. 涨跌幅颜色错误（红跌绿涨）→ 违反A股习惯，严重误导
5. 数据库无法写入（回测结果丢失、信号丢失）→ 核心功能不可用

### 5.2 🟠 High（需要修复）
1. 股票名称大面积显示为代码（非退市股）→ 用户体验差
2. 技术形态名称英文（非专业缩写）→ 普通用户无法理解
3. 信号 BUY/SELL 未翻译 → 信号含义不清
4. 页面打不开/404 → 功能不可用
5. 空状态无提示 → 用户不知道是否加载失败

### 5.3 🟡 Medium（建议优化）
1. 价格浮点精度问题（10.200000000000001）→ 显示不美观
2. 构建时间超过5s → 开发体验问题
3. 无中文字体配置 → 可能字体回退
4. 色盲不友好 → 无障碍问题

### 5.4 🟢 Low（几乎不影响）
1. 专业缩写英文（MA5/MACD/KDJ/RSI/BOLL/F10）→ 行业通用，无需翻译
2. SuperTrend 未翻译 → 专业术语，可保留
3. 策略名称英文 → 策略名称可保留英文

## 六、排查流程（A股版）

### 步骤1: 确认当前日期是否是交易日
- 如果是周末/节假日 → 最新数据日期=最近交易日，不算延迟
- 如果是交易日且当前已过 15:00（收盘后）→ 检查数据是否已更新到当天收盘
- 如果是交易日且当前在盘中 → 最新数据日期=最近一个已收盘的交易日，不算延迟

### 步骤2: 检查大盘数据是否正常
- 上证指数、深证成指、创业板指是否返回
- 涨跌幅是否合理（±0.5%~±5%正常）
- 涨跌幅颜色是否正确（红涨绿跌）

### 步骤3: 检查个股数据
- 选取1-2只知名股票（如000001平安银行、600519茅台）
- 检查价格是否合理（茅台1000+正常，平安银行10+正常）
- 检查名称是否正确显示（排除退市股）
- 检查成交量是否合理（不能为0或负数）

### 步骤4: 检查信号数据
- 信号日期是否在最近交易日附近
- 如果是周末，延迟1-2天正常
- 检查信号类型是否翻译（BUY→买入, SELL→卖出）

### 步骤5: 检查技术形态
- 形态名称是否中文（如"头肩顶"而非"head_shoulder_top"）
- 技术指标缩写（MA5/MACD/KDJ等）可以接受

### 步骤6: 检查前端体验
- 涨跌幅颜色: 红涨绿跌 ✓
- 成交量单位: 万/亿 ✓
- 价格格式: 2位小数 ✓
- 页面标题: 中文 ✓
- 空状态提示: 中文 ✓
