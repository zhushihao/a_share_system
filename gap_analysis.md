# Quant Workbench vs 通达信能力差距分析

> 分析时间：2026-06-27
> 对标标杆：通达信（D:\TDX）主流个人炒股终端
> 目标：能力层面先对齐，再个性化优化

---

## 一、数据层差距（最严重）

| 能力 | 通达信 | 当前系统 | 差距 | 优先级 |
|------|--------|----------|------|--------|
| **实时数据优先级** | 实时 > 离线 | 实时尝试失败→降级离线 | 实时接口不稳定，降级逻辑反了 | P0 |
| **数据持久化** | 实时数据自动写入本地 | 实时数据不持久化 | 每次重启从零开始 | P0 |
| **实时vs离线比对** | 自动比对差异 | 无 | 数据不一致无感知 | P0 |
| **数据下载** | 自动下载历史K线 | 手动触发 | 无自动数据同步 | P1 |
| **分钟级数据** | 1/5/15/30/60分钟 | 有分钟API但不完整 | 分钟级数据聚合不完整 | P1 |
| **分时图** | 当日Tick级分时 | 无 | 完全缺失 | P1 |
| **分笔成交** | Level-2分笔 | 无 | 完全缺失 | P2 |

## 八、核心修复路线图

### 第一阶段：数据底座（P0，本周完成）
1. 实时数据优先化 + 实时数据写入SQLite
2. 每日收盘后自动比对实时 vs TDX离线数据
3. 大盘数据端点（上证/深证/创业板）
4. 数据平台 Dashboard（数据健康/覆盖/时效性）

### 第二阶段：个股体验（P1，下周完成）
1. 分时图组件 + 分钟级数据接口
2. 个股F10基本信息（总股本/市值/行业）
3. 五档行情（买卖盘）
4. 个股搜索（代码/名称/拼音）

### 第三阶段：大盘与板块（P1，两周完成）
1. 大盘指数实时行情面板
2. 涨跌家数/涨跌比统计
3. 行业板块/概念板块数据
4. 板块涨幅排行

### 第四阶段：图形化增强（P2，后续）
1. 指标叠加到K线图
2. 画线工具（手动趋势线/支撑/阻力）
3. 多图对比
4. 更多技术指标（CCI/WR/VWAP等）
