# Quant Workbench v2.0 系统架构设计

> 以「数据完备 → 分析可拓展 → 买卖点明确」为核心目标

---

## 一、用户核心需求拆解

| 需求 | 当前状态 | 目标状态 | 差距 |
|------|----------|----------|------|
| **完备准确的数据** | 6周期K线+实时行情，有缓存层 | 多源校验、零假数据、分钟级自动刷新 | 缺少数据多源交叉验证 |
| **友好的前端界面** | 基础K线图+指标面板+副图 | 专业级交互、画线工具、热力图、盘口 | 缺少交互式画线、盘口数据 |
| **可拓展的技术分析体系** | 指标+形态+量价+波浪（分散） | 插件化架构、新能力即插即用 | 缺少统一的分析编排层 |
| **买卖点** | 10个信号策略（独立运行） | 多因子合成、置信度评分、入场/出场/止损/止盈 | 缺少买卖点合成引擎 |

## 三、当前系统与目标差距

### 差距1：数据多源校验（完备性）

| 数据源 | 数据类型 | 当前状态 | 缺失 |
|--------|----------|----------|------|
| 通达信离线 | K线（日/周/月/分钟） | ✅ 有 | 滞后1-3天 |
| mootdx Quotes | 实时行情/分钟K线 | ✅ 有 | 分钟线可能不稳定 |
| 东方财富 | 涨停数据 | ✅ 有 | 仅涨停 |
| 财联社 | 实时新闻/电报 | ❌ 无 | 舆情数据缺失 |
| 同花顺/雪球 | 基本面数据 | ❌ 无 | PE/PB/ROE等缺失 |

**解决方案：** 引入多源交叉验证，当实时与离线数据差异>5%时自动告警

### 差距2：分析编排层（可拓展性）

| 当前 | 目标 | 差距 |
|------|------|------|
| 各分析引擎独立运行，无统一调度 | AnalysisOrchestrator 统一编排 | 缺少编排器 |
| 新增分析能力需改多处代码 | 插件化：新增Pipeline即可 | 缺少插件接口 |
| 前端需发多个请求 | 统一API：1个请求获取所有分析 | 缺少统一接口 |

**解决方案：** 建立 `AnalysisOrchestrator` + `AnalysisPipeline` 基类，新能力只需实现基类

### 差距3：买卖点引擎（核心目标）

| 当前 | 目标 | 差距 |
|------|------|------|
| 10个独立策略，各跑各的 | 多因子合成引擎 | 缺少合成层 |
| 信号只有「买入/卖出/关注」 | 信号含：入场价、止损价、止盈价、仓位建议 | 缺少交易参数 |
| 无置信度评分 | 0-100%置信度 + 多因子分解 | 缺少评分体系 |
| 无止损/止盈建议 | 基于ATR/支撑阻力的动态止损 | 缺少风控参数 |

**解决方案：** 建立 `SignalComposer` 多因子合成引擎，输出完整交易计划

### 差距4：前端交互（友好性）

| 当前 | 目标 | 差距 |
|------|------|------|
| 基础K线图 | 专业级交互图表（类似TradingView） | 缺少画线工具 |
| 指标纯数值展示 | 指标叠加到K线图 | 缺少主图叠加 |
| 无买卖点标记 | K线图上标记买入/卖出信号 | 缺少信号标记 |
| 无盘口数据 | 买卖五档/逐笔成交 | 缺少盘口 |
| 无热力图 | 板块/个股涨跌热力图 | 缺少可视化 |

**解决方案：** 升级 TradingViewChart，添加画线工具、买卖点标记、盘口组件

## 四、实施路线图

### 第一阶段：数据完备（1-2天）

**目标：零假数据，多源校验，自动刷新**

1. 数据多源交叉验证
   - 实时行情 vs 离线K线最新数据比对
   - 差异>5%时自动告警，标记为"可疑数据"
   - 优先使用实时数据，离线作为降级兜底

2. 数据质量增强
   - 增加「数据源一致性」维度检查
   - 增加「价格合理性」检查（涨跌停限制）
   - 增加「成交量异常」检查（突然放大/缩小）

3. 自动刷新机制
   - 分钟级数据：交易时段每30秒刷新
   - 日级数据：收盘后自动刷新
   - 指数数据：实时刷新（5分钟缓存）

### 第二阶段：分析编排层（3-5天）

**目标：插件化架构，统一API，能力链复用**

1. 建立 `AnalysisPipeline` 基类
```python
class AnalysisPipeline(ABC):
    @abstractmethod
    def analyze(self, df: pd.DataFrame, context: Dict) -> AnalysisResult:
        pass
    
    @property
    @abstractmethod
    def name(self) -> str:
        pass
    
    @property
    @abstractmethod
    def dependencies(self) -> List[str]:
        """声明依赖的上游Pipeline"""
        return []
```

2. 将现有分析能力改造为Pipeline
   - `IndicatorPipeline`（已有）
   - `PatternPipeline`（新建）
   - `VolumeAnalysisPipeline`（新建）
   - `SupportResistancePipeline`（新建）
   - `WavePipeline`（新建）

3. 建立 `AnalysisOrchestrator`
   - 依赖图解析：按依赖顺序执行Pipeline
   - 缓存管理：每个Pipeline的结果独立缓存
   - 统一输出：合并所有Pipeline结果为 `AnalysisReport`

4. 统一API：`GET /api/v1/analysis/{symbol}?period=daily`
   - 1个请求返回所有分析结果
   - 前端7请求→1请求

### 第三阶段：买卖点引擎（5-7天）

**目标：多因子合成，输出完整交易计划**

1. 建立 `SignalComposer` 引擎
```python
class SignalComposer:
    """多因子合成引擎"""
    
    def compose(self, analysis_report: AnalysisReport) -> TradingSignal:
        # 1. 收集各因子评分
        factor_scores = {
            "indicator": self._score_indicator(analysis_report.indicators),
            "pattern": self._score_pattern(analysis_report.patterns),
            "volume": self._score_volume(analysis_report.volume_analysis),
            "support_resistance": self._score_sr(analysis_report.support_resistance),
            "wave": self._score_wave(analysis_report.wave),
        }
        
        # 2. 加权合成
        weights = {"indicator": 0.3, "pattern": 0.25, "volume": 0.25, "support_resistance": 0.15, "wave": 0.05}
        total_score = sum(factor_scores[k] * weights[k] for k in factor_scores)
        
        # 3. 生成交易信号
        if total_score > 0.7:
            return TradingSignal(
                type="BUY",
                confidence=total_score,
                entry_price=analysis_report.ohlcv.iloc[-1]["close"],
                stop_loss=self._calculate_stop_loss(analysis_report),
                take_profit=self._calculate_take_profit(analysis_report),
                position_size=self._calculate_position(total_score),
                rationale=self._build_rationale(factor_scores),
                factors=factor_scores,
            )
        elif total_score < -0.7:
            return TradingSignal(
                type="SELL",
                confidence=abs(total_score),
                ...
            )
```

2. 各因子评分逻辑
   - **指标因子**：MA多头排列、MACD金叉、KDJ超卖区金叉 → 正向评分
   - **形态因子**：头肩底/双底/三角形突破 → 正向评分；头肩顶/双顶 → 负向评分
   - **量价因子**：放量突破/底背离 → 正向；顶背离/放量滞涨 → 负向
   - **支撑阻力因子**：价格接近强支撑+反弹 → 正向；接近强阻力+回落 → 负向
   - **波浪因子**：5浪末端+调整浪开始 → 负向；调整浪结束+5浪开始 → 正向

3. 止损/止盈计算
   - 止损：基于ATR（平均真实波幅）× 2倍，或最近支撑位下方
   - 止盈：基于R:R（风险收益比）1:2 或 1:3，或阻力位

4. 新API：`GET /api/v1/analysis/{symbol}/signal?period=daily`
   - 返回完整交易信号：类型/置信度/入场/止损/止盈/仓位/理由

### 第四阶段：前端升级（5-7天）

**目标：专业级交互，买卖点可视化**

1. **K线图升级**
   - 主图：MA/BOLL 叠加（已有）+ 买卖点标记
   - 画线工具：趋势线、水平线、黄金分割线（用户可手绘）
   - 形态标记：自动显示识别出的形态（已有）+ 买卖点标记

2. **买卖点标记**
   - 买入信号：K线下方绿色箭头 + 价格标签
   - 卖出信号：K线上方红色箭头 + 价格标签
   - 止损线：红色虚线水平线
   - 止盈线：绿色虚线水平线

3. **信号面板**
   - 当前信号卡片：类型/置信度/入场/止损/止盈
   - 因子分解雷达图：各因子评分可视化
   - 历史信号列表：按日期排序，显示盈亏结果

4. **盘口组件**（如果数据源支持）
   - 买卖五档 + 逐笔成交（需要实时Level-2数据）
   - 若数据源不支持，则显示「资金流向」（大单/中单/小单）

5. **热力图**
   - 板块涨跌热力图（已有热点板块基础上升级）
   - 个股涨跌热力图（按行业/概念分组）

### 第五阶段：持续优化（持续）

1. **信号质量跟踪**
   - 记录每个信号的实际表现（入场→出场盈亏）
   - 计算信号胜率、盈亏比、夏普比率
   - 自动调整因子权重（基于历史表现）

2. **新分析能力插件**
   - 自定义Pipeline接口：用户可编写Python插件
   - 社区共享：信号策略的分享和下载

3. **数据扩展**
   - 接入更多数据源（东方财富、同花顺、雪球）
   - 基本面数据（PE/PB/ROE/营收增长）
   - 舆情数据（新闻情绪、社交媒体热度）

## 五、关键设计决策

### 决策1：买卖点引擎的权重分配

```
买入信号权重：
  技术指标（30%）：MA/KDJ/MACD/RSI 的综合趋势判断
  形态识别（25%）：头肩底/双底/三角形突破等反转/持续形态
  量价分析（25%）：放量突破/底背离/天量地价等
  支撑阻力（15%）：价格接近强支撑+反弹确认
  波浪结构（5%）：辅助判断浪型位置（5浪末端慎用）

卖出信号权重：
  同上，但方向相反
  额外增加：跌破止损位（100%权重，强制卖出）
```

### 决策2：止损/止盈策略

| 策略 | 触发条件 | 适用场景 |
|------|----------|----------|
| ATR止损 | 入场价 - 2×ATR | 波动较大的股票 |
| 支撑止损 | 最近支撑位下方3% | 有明显支撑的股票 |
| 百分比止损 | 入场价 - 5% | 通用场景 |
| 移动止损 | 盈利后上移止损至成本价+1% | 保护利润 |
| 目标止盈 | 入场价 + 2×(入场价-止损价) | 固定R:R=1:2 |
| 阻力止盈 | 到达最近阻力位 | 有明显阻力的股票 |

### 决策3：信号置信度分级

| 置信度 | 等级 | 建议仓位 | 说明 |
|--------|------|----------|------|
| 90-100% | ★★★★★ | 满仓 | 多因子共振，高概率 |
| 70-89% | ★★★★ | 半仓 | 主要因子共振 |
| 50-69% | ★★★ | 1/3仓 | 部分因子支持 |
| 30-49% | ★★ | 观察 | 弱信号，等待确认 |
| <30% | ★ | 忽略 | 无明确信号 |
