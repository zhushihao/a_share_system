## 自检范围

### 1. 数据质量自检

**API 端点验证**（必做）：
- [ ] 调用 `/api/v1/quote/{symbol}/signal?period=daily` 验证买卖点API
- [ ] 调用 `/api/v1/quote/{symbol}/patterns?period=daily` 验证形态识别API
- [ ] 调用 `/api/v1/quote/{symbol}/volume-analysis?period=daily` 验证量价分析API
- [ ] 调用 `/api/v1/quote/{symbol}/support-resistance?period=daily` 验证支撑阻力API
- [ ] 检查前端构建产物是否存在 `frontend_react/dist/index.html` 及对应 assets

**API 数据质量检查**：
- [ ] 信号 API：HOLD 信号时 stop_loss 是否为 0；BUY 信号时 stop_loss < entry_price；SELL 信号时 stop_loss > entry_price
- [ ] 信号 API：confidence 是否在 0.5-1.0 之间（阈值已从 0.7 降至 0.5）
- [ ] 支撑阻力 API：support 价格是否全部低于当前 close；resistance 价格是否全部高于当前 close
- [ ] 量价分析 API：divergence_count 是否过大（>100 需警惕，>200 视为严重问题）
- [ ] 形态 API：pattern 字段是否被正确映射（前端期望 `pattern` 而非 `type`）
- [ ] 信号 API 的 pattern factor：detected 列表中是否还有重复形态（同一类型多次出现）
- [ ] 信号 API 的 pattern factor：score 是否被 clip 到 -1.0 或 +1.0（过度极端需检查）
- [ ] 信号 API 的 volume factor：是否不再系统性偏空（修复 idxmax  bug 后验证）

## 迭代重点（当前版本 v15）

当前系统已实现：
- 多周期K线数据（分钟/日/周/月/季/年）
- 技术指标（MA/KDJ/MACD/RSI/BOLL）+ 叠加
- 形态识别（双顶/双底/头肩/三角形/V型/斐波那契）+ 评分去重
- 量价分析（放量突破/缩量回调/背离/天量/地量）+ 合理背离检测（v15）
- 支撑阻力（历史高低点/密集成交区/整数关口）+ 字段兼容
- 波浪结构（5-3艾略特波浪）
- 多因子买卖点合成引擎（SignalComposer）+ 形态去重 + 支撑阻力兼容 + 阈值0.5（v15）
- 前端信号面板（置信度/入场/止损/止盈/仓位/因子分解/多空力量对比）+ HOLD筛选（v15）
- 前端K线图买卖点标记（蓝买/橙卖/红止损/绿止盈）+ 统一标记设置（v15）
- 追踪止损（价格移动1R后上移至保本价）
- 信号防抖（5日 Debounce）+ 作用域修复（v15）
- 信号保存到数据库（含 HOLD 诊断）+ 错误日志（v15）
