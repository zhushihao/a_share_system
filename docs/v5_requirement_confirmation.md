## 3. Phase 3 需求描述

### 3.2 高级触发器（新增）

#### 3.2.1 价格触发器（PriceTrigger）

**需求**：
- 监控用户配置的关注列表（watchlist），轮询获取实时行情
- 检测条件：
  - 价格突破：当前价格较前一日收盘价涨幅超过阈值（默认5%）
  - 价格暴跌：当前价格较前一日收盘价跌幅超过阈值（默认-5%）
  - 成交量异动：当日成交量较前5日平均成交量超过倍率（默认2倍）
- 检测到条件后，通过 EventBus 发布 PRICE_BREAKOUT / PRICE_DROP / VOLUME_SPIKE 事件
- 每个股票每个事件类型每天只触发一次（避免重复告警）

**配置参数**（events.yaml）：
```yaml
triggers:
  price:
    enabled: true
    interval_seconds: 30          # 轮询间隔
    watchlist: []                 # 监控股票列表
    breakout_threshold: 0.05      # 突破阈值
    drop_threshold: -0.05         # 暴跌阈值
    volume_spike_ratio: 2.0       # 成交量异动倍率
```

**文件**：`events/price_trigger.py`

#### 3.2.2 板块触发器（SectorTrigger）

**需求**：
- 每5分钟检测一次板块涨跌幅
- 检测条件：
  - 板块突涨：板块整体涨幅超过阈值（默认3%）
  - 板块突跌：板块整体跌幅超过阈值（默认-3%）
  - 板块龙头切换：板块内涨幅第一的股票发生变化
- 检测到条件后，发布 SECTOR_SURGE / SECTOR_DROP / SECTOR_LEADER_CHANGE 事件
- 每个板块每天每种事件只触发一次

**配置参数**：
```yaml
triggers:
  sector:
    enabled: true
    interval_seconds: 300         # 5分钟
    surge_threshold: 0.03         # 板块涨幅阈值
    drop_threshold: -0.03         # 板块跌幅阈值
```

**文件**：`events/sector_trigger.py`

#### 3.2.3 系统监控触发器（SystemTrigger）

**需求**：
- 每60秒检测一次系统健康状态
- 检测项：
  - 内存使用率超过阈值（默认80%）
  - 磁盘使用率超过阈值（默认90%）
  - 数据源不可用（离线+实时均失败）
- 检测到异常后，发布 SYSTEM_ERROR / DATA_SOURCE_DOWN 事件
- 异常恢复后，发布 DATA_SOURCE_RECOVER 事件

**配置参数**：
```yaml
triggers:
  system:
    enabled: true
    interval_seconds: 60
    memory_threshold: 80          # 内存使用率%
    disk_threshold: 90            # 磁盘使用率%
```

**文件**：`events/system_trigger.py`

#### 3.2.4 事件处理器增强

**新增处理器**：
- `PriceAlertHandler`：价格告警时推送通知（记录日志，可扩展飞书推送）
